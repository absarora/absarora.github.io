Classes Vs. ID's

Intro

Within HTML (Hypertext Markup Language), we need ways to structure the elements. The base template for HTML that we use is as follows:

<!DOCTYPE html> # set the document type
<html>
  <head>
  </head>
  <body>
  </body>
</html>

Within the base template, we use more specific elements to style our page. For example:

<h1></h1> # heading tag

<p></p> # paragraph tag

<ol></ol> # ordered list

Styling can be done either within the HTML document using the <style></style> tags or by using a reference link to a CSS file. CSS (a.k.a Cascading stylesheets) let us style the HTML elements. Since, it is likely that we will be using the tags listed above several times repeatedly within our HTML, we may want to style that same element differently. This is where ID’s and Classes comes in.

ID's

ID's are unique. Each element can link with only one unique ID. For example:

<header>
  <h1 id="name">Ajaybir Singh</h1>
</header>

<body>
  <h1 id="title">Devbootcamp’s Experience</h1>
</body>

There are two <h1> tags in the example above with a unique ID of their own. The primary purpose of an ID is to be able to style that specific element in a unique way.

Classes

Classes are not Unique. We can use the same class on various different elements. For example:

<section>
  <article class="grid"></article>
  <article class="grid"></article>
  <article class="grid"></article>
  <article class="grid"></article>
</section>

In the example above, we have four <article> tags with the same class name. The purpose of using classes is to add the same style to the elements that pertain to that class.
